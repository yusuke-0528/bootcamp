・github-flowについて
GitHub Flowとは、GitHubでの開発におけるシンプルで実践的なワークフローの一つである。この手法では、mainブランチを常にデプロイ可能な状態に保ちつつ、機能ごとに小さな単位でブランチを作成し、変更を加えたのちにPull Request（以降はPR）を用いてレビュー・マージを行う。作業は基本的にすべてローカルブランチで行い、mainには直接コミットせず、PRを通じて反映させることが原則である。これにより、変更履歴を明確に保ちつつ、安全かつ迅速に機能開発や修正を進めることが可能となる。GitHub Flowは、複雑なリリース管理や複数の開発段階に対応する「git-flow」とは異なり、新しい機能の追加、既存のコードの修正、バグの修正、ファイル構成の整理など短期間かつ小規模な単位で完結する作業に適している。具体例としては、1つのページにボタンを追加する、変数名を変更する、READMEファイルを更新するなどの作業が繰り返される開発において、毎回の変更をブランチで管理し、レビューを通じて質を保ちつつ必要に応じてすぐに反映・修正できる体制を提供するのが「github-flow」である。
