・git branch
「git branch」はリポジトリ内で初期設定として生まれる「main」以外のブランチを作成するためのコマンドである。
「git branch <ブランチ名>」が基本構造であり<ブランチ名>で名称を決める。ブランチ名の前にコードを入れることでオプションを作る。
「git branch」のオプションの代表例は以下のものがある。

・-a
意味：ローカルブランチとリモートブランチのすべてのブランチを表示する。

・-r
意味：リモートブランチのみ表示する。

・-v
意味：各ブランチの最新コミットメッセージを表示。

・--merge
意味：名前付きコミットからチップコミットに到達できるブランチのみが一覧表示される。

・--no-merge
意味：名前付きコミットにマージされていないブランチのみが一覧表示される。引数がない場合はデフォルトで「HEAD」になる。

・-d <ブランチ名>
意味：指定したブランチを削除する
例：git branch old_html

・-m <元ブランチ名> <新ブランチ名>
意味：ブランチ名の名前を付けなおすことができる。
例：git branch old_html new_html
