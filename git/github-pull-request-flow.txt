・pull requestの流れ
GitHubのPull Request（以降：PR）は、開発者が自身の作業ブランチの変更をmainブランチなどに取り込んでもらうための提案のこと。PRでは、変更内容の差分に加えてその背景や意図を説明し、レビューを依頼する。一般的な流れは、まず作業を行うためのブランチを作成し、変更を加えて「commit」・「push」後、GitHub上でPRを作成、必要に応じて修正対応し、レビューを経てmainブランチへ「merge」する。PRの利点は、チームでコードを確認し合うことでミスを防ぎ、履歴が明確になる点である。また、コメント機能により改善の検討がしやすくコードの質の改善につながる。GitHubではこのPRを中心に開発を進めるスタイルが基本となっており、個人の作業がmainに取り込まれる前に適切なレビューを経ることで、チーム全体での開発効率を上昇させたりミスの見落としの減少につながっていく。
